# Volgograd Time Quest (Mobile Web AR)

Одностраничный сайт для мобильного квеста (iOS/Android) с AR-сканером на **AR.js (A‑Frame)** и barcode‑метками **ID 0–4**.

## Как запустить

### Локально (для проверки)

```bash
python3 -m http.server 8080
```

Открой `http://localhost:8080` на телефоне (если вы в одной Wi‑Fi сети, можно открыть по IP компьютера).

### Важно про HTTPS

Камера в мобильных браузерах почти всегда требует **HTTPS**.

- **Для реального использования**: залейте папку проекта на GitHub Pages / Netlify / любой HTTPS‑хостинг.
- **Для теста**: `localhost` обычно разрешён.

## Где метки (AR points)

Чистые метки (лучше для распознавания):

- `assets/markers/marker_0.png` … `marker_4.png`

Версии “для печати” с подписью (удобно распечатать/показать на 2‑м телефоне):

- `assets/markers/printable_0.svg` … `printable_4.svg`

Рекомендации:
- печать **8–10 см** по стороне
- ровно, без бликов, без сильных теней
- держать плоско (не сгибать лист)

Для 3D солдата Второй мировой используйте:
- `assets/markers/marker_1.png` (Мамаев курган)
- `assets/markers/marker_2.png` (Дом Павлова)

## Как сделать “голографического солдата” (2 варианта)

### Вариант A (самый стабильный) — 2D “hologram sprite”

Это то, что уже использовано в проекте: персонаж — это **картинка (SVG/PNG/WebP)**, которая появляется поверх маркера в AR.

1. Нарисуйте персонажа (например, советского солдата) в Figma/Illustrator/Photoshop.
2. Экспортируйте в `PNG` (512×512 или 1024×1024) с прозрачным фоном.
3. Положите файл в `assets/characters/`, например:
   - `assets/characters/ussr_soldier_custom.png`
4. В `index.html` найдите массив `STOPS` и замените путь `sprite` у нужной точки:
   - было: `assets/characters/ussr_soldier.svg`
   - станет: `assets/characters/ussr_soldier_custom.png`
5. Чтобы был “голографический” вид:
   - делайте основной цвет персонажа ближе к **cyan/blue**
   - оставляйте внутри силуэта “светлые линии/контур”
   - избегайте мелких деталей — на камере они теряются

### Вариант B (более “настоящая голограмма”) — 3D модель glTF

Если хотите 3D‑солдата:

1. Сделайте модель в Blender (можно очень простую low‑poly).
2. Материал: **Emissive** (светящийся) + голубой цвет, без сложных PBR.
3. Экспорт в `glTF (.glb)`.
4. Подключение в A‑Frame (внутри `<a-marker ...>`):
   - `<a-asset-item id="soldier" src="assets/models/soldier.glb"></a-asset-item>`
   - `<a-entity gltf-model="#soldier" scale="0.6 0.6 0.6" position="0 0 0"></a-entity>`

Важно: 3D на мобильном часто тяжелее и менее предсказуем по FPS, поэтому для квеста “на улице” обычно лучше вариант A.

## Текущая настройка в этом проекте

- 3D модель солдата подключена из `assets/models/ww2_soviet_uniform.glb` для точек 2 и 3.
- Озвучка солдата: `assets/audio/soldier_ru.mp3` (русский).
- Если `soldier_ru.mp3` не найден, используется fallback через браузерный синтез речи (ru-RU).
